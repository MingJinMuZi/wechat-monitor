## 阅读数据获取方案

### 方案1: 免费第三方平台（推荐）

| 平台 | 费用 | 限制 | 链接 |
|------|------|------|------|
| **新榜** | 免费 | 有限额 | https://newrank.cn |
| **极致了** | 免费 | 有限额 | https://www.jzl.com |
| **次幂数据** | 免费 | 有限额 | https://www.cimidata.com |

**使用方式**:
1. 注册账号
2. 搜索公众号
3. 查看文章数据
4. 手动录入到系统

```bash
# 手动录入数据
node scripts/add-article-data.js "链接" 10000 500 200 100 50
```

### 方案2: 开源Hook方案（免费但复杂）

**项目**: wechat-spider (GitHub开源)

**费用**: 免费

**要求**:
- 额外服务器（2G内存+）
- MySQL数据库
- Redis缓存
- 代理服务器配置
- SSL证书配置
- 持续维护

**部署复杂度**: ⭐⭐⭐⭐⭐

**维护成本**: 高（微信更新可能导致失效）

**适用场景**: 大量账号监控、自动化需求高

### 方案3: 当前方案（推荐）

**已支持**:
- ✅ 完整文字内容
- ✅ 完整图片素材
- ✅ HTML/Markdown导出
- ⚠️ 阅读数据需手动补充

**补充方式**:
1. 新榜/极致了查看
2. 手动录入数据
3. 生成完整报告

## 推荐方案

### 短期（当前）

使用现有功能：
```bash
# 1. 抓取文章（文字+图片）
node scripts/fetch-article-full.js "链接"

# 2. 新榜查看阅读数据
# 3. 手动录入数据
node scripts/add-article-data.js "链接" 10000 500 200 100 50

# 4. 生成报告
node scripts/generate-report.js "公众号名"
```

### 中期（如需自动化）

方案A: 申请第三方API免费额度
- 新榜/极致了注册
- 申请API Key
- 配置自动获取

方案B: 部署开源Hook
- 需要额外服务器
- 部署wechat-spider
- 配置MySQL+Redis

### 长期（大量数据）

部署完整Hook服务：
- 专用服务器
- wechat-spider全套
- 自动化监控

## 决策建议

| 需求 | 推荐方案 | 成本 |
|------|---------|------|
| 少量分析 | 当前方案+手动录入 | 免费 |
| 中等规模 | 第三方平台免费版 | 免费（有限额）|
| 大规模自动化 | 部署Hook服务 | 服务器费用 |

**当前推荐**: 使用现有方案，阅读数据通过新榜/极致了查看后手动录入。

**理由**:
1. 文字+图片已完整抓取
2. 阅读数据不是核心分析要素
3. 手动录入成本低于部署Hook服务
4. 第三方免费版足够日常使用
