# WeChat Monitor - 报告输出标准

## 报告结构标准

### 一级结构
```
一、数据采集
二、抓取文章  
三、深度分析报告
```

### 二级结构

#### 一、数据采集
- 1.1 账号信息（表格）
- 1.2 数据统计（表格）
- 1.3 采集方式（表格）

#### 二、抓取文章
- 2.1 文章基本信息（表格）
- 2.2 文章素材（图片列表）
- 2.3 阅读数据（表格）⭐新增
- 2.4 文章正文（代码块）

#### 三、深度分析报告
- 3.1 标题套路拆解（3列表格：组成部分+内容+作用）
- 3.2 标题维度分析（表格）
- 3.3 内容结构（4列表格：部分+内容+字数+作用）
- 3.4 语言特点（多表格：维度特征+词汇分类+修辞手法）
- 3.5 互动策略（表格）
- 3.6 爆款特征（表格）
- 3.7 对标价值（表格）
- 3.8 总结评分（表格）

## 输出格式标准

### 标题层级
```markdown
# 一级标题（报告主标题）
## 二级标题（章节标题）
### 三级标题（小节标题）
#### 四级标题（细分标题）
```

### 强调标记
- **粗体**：关键词、重点
- `代码`：链接、文件名、路径
- ⭐：评分星星（1-5个）
- ✅/❌：状态标记

### 分隔线
```markdown
---
```

```markdown
...

...

## 对比分析
...

...
```

### 批量输出方式
1. **单文件多账号**：所有账号在一个文件
2. **多文件单账号**：每个账号单独文件
3. **索引+详情**：索引文件+详情文件链接

## 文件输出流程

```javascript
// 1. 生成报告内容
const report = generateReport(article);

// 2. 写入文件
const filename = `${account}-${date}-report.md`;
fs.writeFileSync(filename, report);

// 3. 返回文件路径
console.log(`报告已生成: ${filename}`);
```
